IoT Based Smart Home System📌 AimThis project aims to create an IoT-based Smart Home System to control and monitor home lights from a remote location using a NodeMCU microcontroller.
🎯 ObjectivesTo control and monitor household appliances from a remote location using the Internet. To switch appliances between manual control and automation. To integrate voice control using Google Assistant or Amazon Alexa. To control multiple appliances using a relay module and NodeMCU.
🛠️ Components RequiredThe following hardware and software components are used in this project:NodeMCU (ESP8266): 1 unit. This acts as the main controller, featuring built-in WiFi and operating at 3.3V to control devices via the internet. LED (Bulb representation): 1 or 2 units. Acts as a bulb or device that glows when powered and is controlled through GPIO pins. Breadboard: 1 unit. Used for temporary circuit connections without the need for soldering. Resistor (2200 – 3302): 2 units. Limits current to protect the LED and prevents it from burning. Micro USB Cable: 1 unit.Jumper Wires: As required. Used alongside the breadboard for temporary circuit connections. Smartphone: 1 unit. Used for remote control.
⚙️ How It WorksThe system operates through seamless cloud integration:Network Connection: The NodeMCU (ESP8266) is connected to a local WiFi network using an SSID and password. Cloud Integration: The device is linked to the Sinric Pro platform, which functions as the cloud server. User Input: The user sends ON/OFF commands from a smartphone via Google Assistant, Alexa, or the Sinric Pro App. Processing: The command travels to the Sinric Pro cloud, and then directly to the NodeMCU via the internet. Execution: The NodeMCU processes the incoming command and controls the designated GPIO pin (such as D1 or D2). Output: The LED connected to that specific pin turns ON or OFF accordingly.
